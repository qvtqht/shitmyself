var cookie = function( name, value, days ){
		// if value is undefined, get the cookie value
		if( value === undefined ){
			var cookiestring = "; " + w.document.cookie;
			var cookies = cookiestring.split( "; " + name + "=" );
			if ( cookies.length === 2 ){
				return cookies.pop().split( ";" ).shift();
			}
			return null;
		}
		else {
			// if value is a false boolean, we'll treat that as a delete
			if( value === false ){
				days = -1;
			}
			var expires = "";
			if ( days ) {
				var date = new Date();
				date.setTime( date.getTime() + ( days * 24 * 60 * 60 * 1000 ) );
				expires = "; expires="+date.toGMTString();
			}
			w.document.cookie = name + "=" + value + expires + "; path=/";
		}
	};

Added Support for phonetic Russian typing in text fields.

This is definitely a "scratching my own itch" feature, primarily intended for desktop.

To ENABLE: Press Alt+` or Alt+R in text field.

To SWITCH BACK TO ENGLISH: same thing, press Alt+` or Alt+R.

If successful, the text box will get a red border at the top when Russian, blue when back to English.

Here is a the layout on a U.S. keyboard:

 Щ 1 2 3 4 5 6 7 8 9 0 ь ъ [Backspc]
[Tab] Я Ш Е Р Т Ы У И О П Ю Ж   Э
[Caps] А С Д Ф Г Ч Й К Л ; ' [Enter]
[Shift] З Х Ц В Б Н М , . /  [Shift]
[Ctl][M][Alt][______][Al][M][m][Ctl]

To type ё, press Alt+E.
To type -/_ and =/+, also use the Alt key.

I copied Apple's Russian Phonetic layout, because I'm most used to it, and it makes the most sense to me.
* I think it is easy to remember that ь and ъ are on the - and + keys.
* Less frequently used letters are placed at the edges.
* Most of the letters are phonetically aligned.
* I also like that most of the punctuation remains unchanged.
* I don't care about traditional Russian punctuation, such as No, <<, >>, etc.

There are a couple of known issues, which I intend to fix in the future:

* The typed letters are always added to the end of the text field, ignoring selection and cursor.
* The letters щ, ь, ъ, ю, ж, э do not obey Caps Lock.
* There is no global setting, it must be turned on each time.
* The feature is available only on Write and Reply fields.
* Only newer browsers, about 2010+, support this feature.

sub GetDigitColor() { # returns a 2-char color that corresponds to a digit for coloring the clock's digits
	# Not sure of purpose, might be useful

	my $digit = shift;

	if (!$digit) {
		return;
	}
	if (length($digit) != 1) {
		return;
	}
	#todo $digit must be ^[0-9]{1}$

	my $digitColor = floor($digit / 10 * 255);
	my $digitColorHex = sprintf("%X", $digitColor);

	if (length($digitColorHex) < 2) {
		$digitColorHex = '0' . $digitColorHex;
	}

	return $digitColor;
}
